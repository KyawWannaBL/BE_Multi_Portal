4) Operation flows (what staff actually does)
Flow 1: Staff login (EN / MY)

User goes to /login

Login by Password or Email Link/OTP

System loads profiles → checks must_change_password

If admin role → enforce MFA (enroll or verify)

Redirect to correct portal: Operations / Execution / Finance / etc

Flow 2: Execution (Rider/Driver/Helper) delivery workflow

Open /portal/execution → see assigned shipments

Open /portal/execution/navigation

Plan route (optimize or directions)

Start navigation + optionally Share location

When arriving in radius → geofence triggers tracking event

Capture POD (signature/photo/OTP) → tracking event inserted

Mark delivered → updates shipment status/time

Flow 3: Branch/Operations live tracking (for branch offices)

Open /portal/operations/tracking

View live riders on map (subscribed via Supabase realtime)

Monitor ETA/remaining distance if published

Use this for branch coordination (handoffs, exceptions, late deliveries)
